Week 12: Struktur Database pada Final Project

Database pada aplikasi MieME menggunakan MySQL sebagai Database Management System (DBMS) dan dikelola melalui mekanisme migration berbasis PHP Native yang dikembangkan sendiri dengan konsep serupa Laravel Migration. Seluruh definisi struktur tabel ditulis dalam bentuk file PHP pada direktori database/migrations, di mana setiap migration memiliki fungsi up() untuk membuat tabel dan down() untuk menghapus tabel (rollback). Proses migrasi dan pengisian data awal dijalankan melalui custom Command Line Interface (CLI) tool db.php, sehingga pembuatan, penghapusan, maupun reset database dapat dilakukan secara otomatis menggunakan perintah seperti php db.php migrate, php db.php rollback, php db.php seed, dan php db.php reset. Pendekatan ini memberikan kemudahan dalam pengelolaan versi database, menjaga konsistensi struktur antar lingkungan pengembangan, serta mendukung arsitektur MVC yang digunakan pada aplikasi. Hasil implementasi menghasilkan 11 tabel utama, yaitu users, categories, menus, cart_items, orders, order_items, payment_methods, payments, reviews, favorites, dan faqs, yang saling berelasi untuk mendukung fitur autentikasi pengguna, katalog menu, keranjang belanja, checkout, pembayaran, unggah bukti transfer, riwayat pesanan, ulasan pelanggan, serta daftar menu favorit. Selain itu, file yang diunggah pengguna seperti bukti pembayaran tidak disimpan pada direktori publik, melainkan pada folder penyimpanan khusus (storage/uploads) sehingga hanya dapat diakses melalui mekanisme aplikasi yang telah dikendalikan oleh sistem untuk meningkatkan keamanan data.
